# TaskTrackerApi

REST API для управления проектами и задачами.

## Технический стек
| Технология | Назначение |
|---|---|
| **ASP.NET Core 8** | Фреймворк |
| **EF Core 9** | ORM |
| **PostgreSQL** | Основное хранилище данных |
| **Redis** | Кэширование списков проектов и задач |
| **JwtBearer** | Стратегия аутентификации |
| **FluentValidation** | Валидация входных данных |
| **AutoMapper** | Маппинг моделей |
| **Serilog** | Логирование |
| **xUnit** | Тестирование |
| **Docker** | Контейнеризация |

## Архитектура

Приложение разделено на три явных слоя, каждый из которых зависит только от нижележащего:
```
Controller -> Service -> Repository -> DbContext
```

Абстрактный `RepositoryBase<TEntity, TKey>` реализует основные операции для сущности (`GetById`, `Add`, `Update`, `Remove` и т.д.), которые наследуются конкретными репозиториями и расширяются при необходимости.

### Кеширование

Списки проектов и задач кешируются в Redis. Ключ кеша включает номер версии, например:
```
projects:{version}:{page}:{pageSize}
```

При выполнении любых операций на изменение данных версия повышается, что делает все существующие записи устаревшими.

### Обработка исключений

В приложении реализован `GlobalExceptionsHandler`, перехватывающий все необработанные исключения.

Для отправки кодов ошибок из сервисов был реализован `AppException(string message, string title, int statusCode)`, от которого наследуются конкретные исключения, такие как `NotFoundException`, `ConflictException` и т.д.
Эти исключения также обрабатываются в глобальном обработчике.

В production среде сообщение об ошибке, включая stacktrace, будет сокращено в целях обеспечения безопасности.

### Валидация

Валидаторы подключены через `AddFluentValidationAutoValidation()`.
Невалидный запрос отклоняется до входа в контроллер, исключение будет перехвачено глобальным обработчиком и возвращается как `400 Bad Request`.

### Миграции

В приложении созданы три миграции, которые автоматически применяются при запуске.

| Наименование | Назначение |
|---|---|
| AddTasks | Создает таблицу задач |
| AddProjects | Создает таблицу проектов |
| AddUser | Создает таблицу пользователей |

## API

### Аутентификация

Проект использует JWT Bearer аутентификацию.

| Метод | Путь             | Тело запроса                 | Описание    |
|-------|------------------|------------------------------|-------------|
| POST  | `/auth/register` | `{ "username", "password" }` | Регистрация |
| POST  | `/auth/login`    | `{ "username", "password" }` | Вход        |

Оба эндпоинта возвращают `{ "token": "...", "expiresAt": "..." }`.

Требования к паролю: минимум 8 символов, содержит хотя бы один спецсимвол.

В базе данных пароль пользователя хранится в виде хэша.

Параметры генерации токена находятся в `appsettings.json`

Защищённые эндпоинты требуют заголовок:
```
Authorization: Bearer <token>
```

### Проекты

| Метод  | Путь             | Авторизация | Описание                       |
|--------|------------------|-------------|--------------------------------|
| GET    | `/projects`      | -           | Список проектов (с пагинацией) |
| GET    | `/projects/{id}` | -           | Проект с задачами              |
| POST   | `/projects`      | требуется   | Создать проект                 |
| PUT    | `/projects/{id}` | требуется   | Обновить проект                |
| DELETE | `/projects/{id}` | требуется   | Удалить проект                 |

Параметры пагинации: `?page=1&pageSize=10`

### Задачи

| Метод  | Путь          | Авторизация | Описание                     |
|--------|---------------|-------------|------------------------------|
| GET    | `/tasks`      | -           | Список задач (с фильтрацией) |
| GET    | `/tasks/{id}` | -           | Задача по ID                 |
| POST   | `/tasks`      | требуется   | Создать задачу               |
| PUT    | `/tasks/{id}` | требуется   | Обновить задачу              |
| DELETE | `/tasks/{id}` | требуется   | Удалить задачу               |

Параметры фильтрации: `?isCompleted=true&projectId=<guid>`

## Запуск

### Локально

**Требования:** .NET 8 SDK, PostgreSQL, Redis

1. Настройте строки подключения в объекте `ConnectionStrings` в `appsettings.json`
```json
"ConnectionStrings": {
  "Postgres": "Host=localhost;Port=5432;Database=tasktracker;Username=postgres;Password=postgres",
  "Redis": "redis:6379"
},
```

2. Запустите приложение
```bash
dotnet run --project TaskTrackerApi
```

3. Swagger доступен по адресу `http://localhost:<port>/swagger`

### Docker

1. Запустите приложение из compose-файла
```bash
docker compose up -d
```

2. Swagger доступен по адресу `http://localhost:8080/swagger`
